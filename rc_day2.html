🚀 아두이노 RC카 교실 2주차 교재: RC카의 '움직임' 이해하기 (모터 기초)

💡 이번 주 학습 목표

  • **부저**를 이용해 아두이노로 다양한 소리를 내는 방법을 배워요.
  • **아날로그 센서**의 원리를 이해하고, **조도 센서**를 이용해 주변 밝기 값을 읽어와요.
  • 아날로그 값을 원하는 범위로 변환하는 **데이터 맵핑(Mapping)** 개념을 익혀요.
  • RC카의 핵심 부품인 **DC 모터**의 작동 원리를 이해하고, **모터 드라이버**를 사용해 제어하는 방법을 배워요.

1회차: 아두이노로 '소리' 내기 & 아날로그 센서 기초

🎶 1. 아두이노가 소리를 내다! (부저)

아두이노는 스피커처럼 복잡한 부품 없이도 간단한 소리를 낼 수 있어요. 바로 **부저(Buzzer)**라는 부품을 이용해서요. 부저는 전기가 통하면 '삐' 소리를 내는 작은 장치예요. 특정 주파수의 전기 신호를 보내면 다양한 음계의 소리를 만들 수도 있답니다. RC카에 경적이나 알림음을 추가할 때 유용해요.

부저의 종류:

  • **피에조 부저(Piezo Buzzer)**: 내부에 발진 회로가 없어 아두이노에서 직접 주파수를 제어해야 해요. 다양한 음계를 낼 수 있습니다. (주로 사용)
  • **액티브 부저(Active Buzzer)**: 내부에 발진 회로가 있어 전원만 연결하면 특정 소리를 내요. 음계 제어는 어렵습니다.

(피에조 부저와 액티브 부저의 실제 사진 또는 그림을 삽입해 주세요. 피에조 부저가 더 많이 사용됨을 강조.)

🔌 2. 회로 연결하기: 부저로 멜로디 연주하기

준비물:

  • 아두이노 우노 보드
  • 브레드보드
  • 피에조 부저 1개
  • 저항 1개 (220Ω 또는 330Ω) - 선택 사항이지만 안정적인 소리를 위해 권장
  • 점퍼선 (수-수)

회로 연결 순서:

  1. **브레드보드 전원 연결**: (1주차와 동일)
    • 아두이노의 **GND 핀**에서 브레드보드의 **파란색 (-) 라인**에 연결합니다.
    • 아두이노의 **5V 핀**에서 브레드보드의 **빨간색 (+) 라인**에 연결합니다.
  2. **부저 연결**:
    • 부저의 **긴 다리(+)** 또는 표시된 (+) 다리를 저항에 연결하고, 저항의 다른 쪽 다리를 아두이노의 **디지털 핀 8번**에 연결합니다.
    • 부저의 **짧은 다리(-)** 또는 표시된 (-) 다리를 브레드보드의 **파란색 (-) 라인(GND)**에 연결합니다.

(아두이노, 브레드보드, 피에조 부저, 저항이 연결된 회로도 이미지를 삽입해 주세요.)

📝 3. 코딩하기: 부저로 '도레미파솔라시도' 연주

아두이노에서 소리를 내기 위해서는 `tone()` 함수를 사용해요. 이 함수는 특정 핀에서 지정된 주파수의 소리를 내도록 합니다.

// 아두이노 RC카 교실 - 2주차 1회차: 부저로 멜로디 연주 예제

const int buzzerPin = 8; // 부저가 연결된 디지털 핀 번호

// 음계별 주파수 정의 (미리 정해진 값들)
#define NOTE_C4  262
#define NOTE_D4  294
#define NOTE_E4  330
#define NOTE_F4  349
#define NOTE_G4  392
#define NOTE_A4  440
#define NOTE_B4  494
#define NOTE_C5  523

void setup() {
  // setup() 함수는 부저 핀을 출력 모드로 설정할 필요가 없습니다.
  // tone() 함수가 내부적으로 핀 모드를 설정합니다.
  Serial.begin(9600);
  Serial.println("멜로디 연주 시작!");
}

void loop() {
  // '도' 소리 내기 (262Hz 주파수, 500밀리초 동안)
  tone(buzzerPin, NOTE_C4, 500); 
  delay(600); // 소리가 끝난 후 다음 소리까지 약간의 간격

  // '레' 소리 내기
  tone(buzzerPin, NOTE_D4, 500);
  delay(600);

  // '미' 소리 내기
  tone(buzzerPin, NOTE_E4, 500);
  delay(600);

  // '파' 소리 내기
  tone(buzzerPin, NOTE_F4, 500);
  delay(600);

  // '솔' 소리 내기
  tone(buzzerPin, NOTE_G4, 500);
  delay(600);

  // '라' 소리 내기
  tone(buzzerPin, NOTE_A4, 500);
  delay(600);

  // '시' 소리 내기
  tone(buzzerPin, NOTE_B4, 500);
  delay(600);

  // '도' (높은 도) 소리 내기
  tone(buzzerPin, NOTE_C5, 500);
  delay(600);

  noTone(buzzerPin); // 현재 핀에서 재생되는 모든 소리를 멈춥니다.
  delay(2000);       // 모든 음이 끝난 후 2초간 대기 (반복을 위해)
}

코딩 용어 상세 설명:

  • `#define NOTE_C4 262`: `#define`은 특정 값을 상수로 정의할 때 사용해요. `NOTE_C4`라는 이름으로 262라는 숫자를 사용하겠다고 미리 정해두는 거죠. 이렇게 하면 코드를 읽기 쉽고 수정하기 편리해요.
  • `tone(핀번호, 주파수, 지속시간_밀리초);`: 지정된 핀에서 특정 **주파수(Hz)**의 소리를 **지정된 시간(밀리초)** 동안 재생하는 함수입니다. 지속시간을 생략하면 `noTone()` 함수를 호출할 때까지 소리가 계속 납니다.
  • `noTone(핀번호);`: `tone()` 함수로 재생되던 소리를 멈추는 함수입니다.

✅ 4. 결과 확인

코드를 아두이노에 업로드하면, 부저에서 '도레미파솔라시도' 멜로디가 연주되는 것을 들을 수 있을 거예요. 성공적으로 소리를 제어했다면 RC카에 다양한 알림음을 추가할 수 있겠죠?

☀️ 5. 아날로그 센서: 빛의 밝기 감지 (조도 센서)

지난주에 디지털 입력(버튼)을 배웠다면, 이번에는 **아날로그 입력**을 배워볼 차례예요. 디지털은 '켜짐' 또는 '꺼짐'처럼 두 가지 상태만 구분하지만, 아날로그는 '밝다', '어둡다', '조금 밝다'처럼 **연속적인 값**을 측정할 수 있어요.

**조도 센서(CDS, Photoresistor)**는 빛의 양에 따라 저항 값이 변하는 센서예요. 빛이 밝으면 저항 값이 작아지고, 어두우면 저항 값이 커져요. 아두이노는 이 저항 값의 변화를 아날로그 신호로 읽어들여 주변의 밝기를 감지할 수 있답니다. RC카가 어두워지면 자동으로 라이트를 켜는 기능 등을 만들 때 활용할 수 있어요.

(조도 센서(CDS)의 실제 사진 또는 그림을 삽입해 주세요.)

🔌 6. 회로 연결하기: 조도 센서 값 읽기

조도 센서는 저항 값이 변하는 부품이므로, 아두이노에서 값을 읽기 위해서는 **분압 회로**를 구성해야 해요. 고정 저항과 조도 센서가 직렬로 연결된 형태입니다.

준비물:

  • 아두이노 우노 보드
  • 브레드보드
  • 조도 센서(CDS) 1개
  • 저항 1개 (10kΩ)
  • 점퍼선 (수-수)

회로 연결 순서:

  1. **브레드보드 전원 연결**: (동일)
    • 아두이노의 **GND 핀**에서 브레드보드의 **파란색 (-) 라인**에 연결합니다.
    • 아두이노의 **5V 핀**에서 브레드보드의 **빨간색 (+) 라인**에 연결합니다.
  2. **조도 센서 연결**:
    • 조도 센서의 한쪽 다리를 브레드보드의 **빨간색 (+) 라인(5V)**에 연결합니다.
    • 조도 센서의 다른 쪽 다리를 브레드보드의 아무 구멍에 꽂습니다.
  3. **고정 저항 연결**:
    • 10kΩ 저항의 한쪽 다리를 조도 센서의 다른 쪽 다리(아두이노 핀에 연결될 쪽)와 **같은 열**에 꽂습니다.
    • 저항의 다른 쪽 다리를 브레드보드의 **파란색 (-) 라인(GND)**에 연결합니다.
  4. **아두이노-센서 연결**:
    • 조도 센서와 10kΩ 저항이 만나는 지점(같은 열)에서 점퍼선을 연결하여 아두이노의 **아날로그 핀 A0번**에 연결합니다.

(아두이노, 브레드보드, 조도 센서, 10kΩ 저항이 연결된 정확한 회로도 이미지를 삽입해 주세요. 분압 회로의 형태를 명확히 보여주세요.)

📝 7. 코딩하기: 조도 센서 값 시리얼 모니터로 확인

조도 센서에서 읽어온 아날로그 값을 시리얼 모니터로 확인해 봅시다.

// 아두이노 RC카 교실 - 2주차 1회차 심화: 조도 센서 값 읽기 예제

const int cdsPin = A0; // 조도 센서가 연결된 아날로그 핀 번호

void setup() {
  // setup() 함수: 아두이노가 시작될 때 한 번 실행됩니다.

  // 시리얼 통신을 시작합니다. (센서 값을 컴퓨터로 확인하기 위함)
  Serial.begin(9600);
  Serial.println("조도 센서 값 읽기 시작!");
}

void loop() {
  // loop() 함수: setup() 이후 무한히 반복 실행됩니다.

  // 1. cdsPin (A0번 핀)의 아날로그 값을 읽어와 변수에 저장합니다.
  //    analogRead() 함수는 0부터 1023까지의 값을 반환합니다.
  int lightValue = analogRead(cdsPin); 

  // 2. 읽어온 조도 센서 값을 시리얼 모니터에 출력합니다.
  Serial.print("조도 센서 값: ");
  Serial.println(lightValue);

  delay(100); // 0.1초마다 값을 읽도록 딜레이를 줍니다.
}

✅ 8. 결과 확인

  1. 코드를 아두이노에 업로드합니다.
  2. 시리얼 모니터를 엽니다.
  3. 조도 센서에 빛을 비추거나 손으로 가려보면서 시리얼 모니터에 출력되는 `lightValue`가 어떻게 변하는지 확인합니다. (빛이 밝을수록 값이 작아지고, 어두울수록 값이 커질 거예요. 또는 반대일 수도 있습니다. 센서와 저항 연결 방식에 따라 달라집니다.)

2회차: DC 모터 제어의 시작 (모터 드라이버 기초)

⚙️ 1. DC 모터란 무엇일까요?

DC 모터(Direct Current Motor)는 RC카의 바퀴를 움직이는 가장 기본적인 부품이에요. 전기를 흘려주면 계속해서 한 방향으로 회전하는 모터입니다. 전기의 방향을 바꾸면 회전 방향도 바꿀 수 있어요. RC카의 전진, 후진을 담당하는 핵심 부품이죠.

(일반적인 DC 모터의 실제 사진 또는 그림을 삽입해 주세요. 기어 모터도 함께 보여주면 좋습니다.)

주의! DC 모터는 아두이노 핀에 직접 연결하면 안 돼요! 모터는 많은 전류를 필요로 하는데, 아두이노 핀에서 공급할 수 있는 전류량은 매우 적기 때문에 아두이노가 고장 날 수 있어요. 그래서 **모터 드라이버**라는 부품이 필요합니다.

🛡️ 2. 모터 드라이버의 역할 (L298N 모듈)

모터 드라이버(Motor Driver)는 아두이노와 모터 사이에서 다리 역할을 해주는 부품이에요. 아두이노의 작은 신호를 받아 모터가 필요로 하는 큰 전류를 공급해 주고, 모터의 회전 방향도 제어할 수 있게 해줘요. RC카에서 모터를 안전하고 효율적으로 제어하기 위해 필수적인 부품입니다.

가장 흔하게 사용되는 모터 드라이버 중 하나는 **L298N 모터 드라이버 모듈**이에요. 이 모듈은 두 개의 DC 모터를 동시에 제어할 수 있습니다.

(L298N 모터 드라이버 모듈의 실제 사진 또는 핀 명칭이 표시된 다이어그램을 삽입해 주세요.)

L298N 모듈 주요 핀:

  • **OUT1, OUT2**: 첫 번째 모터 연결 (A 모터)
  • **OUT3, OUT4**: 두 번째 모터 연결 (B 모터)
  • **ENA, ENB**: 모터 A, B의 속도 제어 (PWM 핀에 연결)
  • **IN1, IN2**: 모터 A의 방향 제어
  • **IN3, IN4**: 모터 B의 방향 제어
  • **+12V (또는 +VCC)**: 모터에 공급할 외부 전원 입력 (보통 6V~12V 배터리)
  • **GND**: 접지 (아두이노 GND와 연결)
  • **+5V**: 아두이노에 5V 전원을 공급할 때 사용 (점퍼 캡 제거 시)

🔌 3. 회로 연결하기: DC 모터 정회전/역회전 제어

이번에는 L298N 모터 드라이버를 사용해서 DC 모터 하나를 정방향, 역방향으로 회전시켜 볼 거예요.

준비물:

  • 아두이노 우노 보드
  • 브레드보드 (선택 사항, 모터 드라이버에 직접 연결 가능)
  • DC 모터 1개 (기어 모터 권장)
  • L298N 모터 드라이버 모듈 1개
  • 9V 건전지 및 건전지 스냅 (또는 6V~12V 외부 전원)
  • 점퍼선 (수-수)

회로 연결 순서:

  1. **모터 드라이버 전원 연결**:
    • 9V 건전지 스냅의 **빨간색 선(+)**을 L298N 모듈의 **+12V (또는 +VCC) 핀**에 연결합니다.
    • 9V 건전지 스냅의 **검은색 선(-)**을 L298N 모듈의 **GND 핀**에 연결합니다.
  2. **아두이노-모터 드라이버 접지 연결**:
    • 아두이노의 **GND 핀**에서 점퍼선을 연결하여 L298N 모듈의 **GND 핀**에 연결합니다. (매우 중요! 접지를 공유해야 합니다.)
  3. **DC 모터 연결**:
    • DC 모터의 두 선을 L298N 모듈의 **OUT1**과 **OUT2** 핀에 연결합니다. (극성 상관없음, 회전 방향만 바뀜)
  4. **아두이노-모터 드라이버 제어 핀 연결**:
    • 아두이노의 **디지털 핀 7번**을 L298N 모듈의 **IN1 핀**에 연결합니다.
    • 아두이노의 **디지털 핀 6번**을 L298N 모듈의 **IN2 핀**에 연결합니다.
    • 아두이노의 **디지털 핀 9번** (PWM 지원 핀)을 L298N 모듈의 **ENA 핀**에 연결합니다. (속도 제어용, 지금은 일단 연결만)

(아두이노, L298N 모터 드라이버, DC 모터, 9V 배터리가 연결된 정확한 회로도 이미지를 삽입해 주세요. 각 핀의 연결을 명확히 보여주세요.)

주의! L298N 모듈의 5V 점퍼 캡이 꽂혀 있는지 확인하세요. 만약 아두이노에 5V 전원을 공급할 목적이 아니라면 이 캡은 그대로 두는 것이 일반적입니다. (아두이노와 모터 드라이버의 GND는 반드시 연결해야 합니다.)

📝 4. 코딩하기: DC 모터 정회전/역회전

이제 코드를 통해 DC 모터의 회전 방향을 제어해 봅시다.

// 아두이노 RC카 교실 - 2주차 2회차: DC 모터 정회전/역회전 제어 예제

// 모터 드라이버 제어 핀 정의
const int motorIn1 = 7; // L298N의 IN1 핀
const int motorIn2 = 6; // L298N의 IN2 핀
const int motorEnablePin = 9; // L298N의 ENA 핀 (PWM 지원 핀)

void setup() {
  // setup() 함수: 핀 모드 설정 및 시리얼 통신 시작

  // 모터 제어 핀들을 출력(OUTPUT) 모드로 설정합니다.
  pinMode(motorIn1, OUTPUT);
  pinMode(motorIn2, OUTPUT);
  pinMode(motorEnablePin, OUTPUT); // ENA 핀도 출력으로 설정

  // 시리얼 통신 시작
  Serial.begin(9600);
  Serial.println("DC 모터 제어 시작!");

  // 초기에는 모터를 정지시킵니다.
  digitalWrite(motorIn1, LOW);
  digitalWrite(motorIn2, LOW);
  analogWrite(motorEnablePin, 0); // ENA 핀을 LOW (0)로 설정하여 모터 정지
}

void loop() {
  // loop() 함수: 모터 동작 반복

  Serial.println("모터 정회전 (앞으로)");
  // 정회전 (예: IN1=HIGH, IN2=LOW)
  digitalWrite(motorIn1, HIGH);
  digitalWrite(motorIn2, LOW);
  analogWrite(motorEnablePin, 255); // ENA 핀을 HIGH (최대 속도)로 설정하여 모터 작동
  delay(3000); // 3초 동안 정회전

  Serial.println("모터 정지");
  // 모터 정지 (IN1, IN2 모두 LOW)
  digitalWrite(motorIn1, LOW);
  digitalWrite(motorIn2, LOW);
  analogWrite(motorEnablePin, 0); // ENA 핀을 LOW (0)로 설정하여 모터 정지
  delay(2000); // 2초 동안 정지

  Serial.println("모터 역회전 (뒤로)");
  // 역회전 (예: IN1=LOW, IN2=HIGH)
  digitalWrite(motorIn1, LOW);
  digitalWrite(motorIn2, HIGH);
  analogWrite(motorEnablePin, 255); // ENA 핀을 HIGH (최대 속도)로 설정하여 모터 작동
  delay(3000); // 3초 동안 역회전

  Serial.println("모터 정지");
  // 모터 정지
  digitalWrite(motorIn1, LOW);
  digitalWrite(motorIn2, LOW);
  analogWrite(motorEnablePin, 0); // ENA 핀을 LOW (0)로 설정하여 모터 정지
  delay(2000); // 2초 동안 정지
}
```

코딩 용어 상세 설명:

  • `digitalWrite(motorIn1, HIGH);`, `digitalWrite(motorIn2, LOW);`: L298N 모듈의 **IN1, IN2 핀**에 HIGH/LOW 신호를 보내 모터의 회전 방향을 결정합니다. IN1, IN2의 조합에 따라 정회전, 역회전, 정지가 가능합니다.
  • `analogWrite(motorEnablePin, 값);`: L298N 모듈의 **ENA (또는 ENB) 핀**에 PWM 신호를 보내 모터의 **속도를 제어**합니다. `0`은 정지, `255`는 최대 속도를 의미합니다.
  • **모터 정지 방법**:
    • `digitalWrite(IN1, LOW); digitalWrite(IN2, LOW);`: 모터에 전압을 공급하지 않아 자연스럽게 정지합니다.
    • `digitalWrite(IN1, HIGH); digitalWrite(IN2, HIGH);`: 모터 양쪽에 같은 전압을 인가하여 **브레이크**를 거는 효과를 줍니다. 더 빠르게 정지할 수 있습니다. (위 예제에서는 간단한 정지 방법을 사용했습니다.)
